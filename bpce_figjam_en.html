Designers
Needs
Visibility and recognition of our work
Knowledge sharing (REX)
Alignment on communication practices
Frustrations
"I build great mockups but nobody reads the specs — or they pick the wrong version."
"It feels like nobody sees what we actually do."
Ideas
Narrative formats (visual REX, posters)
Standardised templates for project docs
Shared design culture (rituals, common tools)
🔗 Communication priorities
Project REX / summary slides
Standardised Figma templates
Shared visual charter
🧑‍💼
PO
Needs
Summaries of UX findings and user pain points
Design + research progress tracking
Decision-oriented formats
Frustrations
"We don't know what you've done or where you are."
"You do things, but we don't know when or how to integrate them."
Ideas
Design newsletter or UX journal
Test / user journey summary sheets
Mini-REX with decision / impact / next step
🔗 Communication priorities
Visual user research summaries
Project REX 3–5 slides
Regular UX journal
🧑‍💻
Devs
Needs
Accessible, reliable, readable specs
Visibility on which DS components are used
Consistency between design and development
Frustrations
"I take Figma screenshots because I don't know which version is right."
"I'm asked for a component that doesn't exist in Neo3 and nobody told me."
Ideas
Project readme or Figma / Confluence one-pager
Regular UX demos in agile rituals
Clear component list with availability status
🔗 Communication priorities
Project readme / Figma one-pager
UX presence in dev rituals
Integrated specs

Leadership / Group
Needs
Consolidated view of design projects
Key data (projects, usage, REX, satisfaction)
Visual business cases for steering
Frustrations
"Hard to follow what design teams are doing — we lack visibility."
"Packaged field feedback is what resonates with leadership."
Ideas
Monthly dashboards
Key design indicators (UX KPIs, deliverables)
High-impact visual infographics
🔗 Communication priorities
UX dashboard
Results infographics
Packaged quarterly / ad hoc REX
